Creating case definitions through the UI
This section will guide you through the process of creating case definitions using the user interface (UI) provided by WKS Platform. Let's explore each step in detail:
1.	Accessing the Case Definition Creation Page:
o	Log in to WKS Platform using your credentials.
o	Navigate to the administration or configuration section of the application.
o	Locate and select the option to create a new case definition. This may be represented by a "New Case Definition" or similar button or link.
2.	Defining Basic Case Definition Information:
o	On the case definition creation page, you will be prompted to provide basic information about the case definition.
o	Enter a unique case definition key that will serve as the identifier for this case definition.
o	Provide a descriptive name and optional description to help identify and differentiate the case definition.
3.	Configuring Case Variables:
o	Identify the data elements or variables that need to be captured and managed within the case.
o	Specify the name, data type, and other relevant attributes for each case variable.
o	Define whether a variable is mandatory or optional for the case.
4.	Creating Case Milestones:
o	Identify the significant milestones or stages that need to be achieved during the case lifecycle.
o	Specify a name, description, and any associated deadlines or conditions for each milestone.
o	Define the order and dependencies between milestones, if applicable.
5.	Designing Case Activities:
o	Identify the tasks, actions, or steps that need to be performed to progress the case.
o	Specify a name, description, and any associated deadlines, forms, or rules for each activity.
o	Define the sequence and dependencies between activities, if applicable.
6.	Configuring Case Forms:
o	Determine the data that needs to be captured at different stages of the case.
o	Design and configure forms to capture and display the required information.
o	Specify the fields, validation rules, and layout for each form.
7.	Defining Case Rules:
o	Identify the decision points or automated actions within the case.
o	Define the conditions and associated actions for each rule.
o	Specify the criteria for evaluating the conditions and triggering the actions.
8.	Setting Case Permissions:
o	Determine the access and actions allowed for different roles or users within the case.
o	Assign permissions for viewing, editing, completing, or performing specific operations on the case.
o	Define the roles or users associated with each permission.
9.	Reviewing and Saving the Case Definition:
o	Review all the configured elements of the case definition.
o	Validate that the information provided is accurate and meets your requirements.
o	Once satisfied, save the case definition to make it available for use in WKS Platform.
Congratulations! You have successfully created a case definition using the WKS Platform UI. Remember to regularly review and update case definitions as your processes evolve. Refer to the WKS Platform documentation or contact your system administrator for further assistance or advanced configuration options.
